1. Linux System Login

Linux systems require authentication to access the system. Users must provide valid credentials to establish a session.
Login Process
login: username
Password: [hidden input]
Last login: Wed Jul 03 10:30:45 2025 from 192.168.1.100

# Example session:
login: tchen
Password:
Last login: Wed Jul 03 10:30:45 2025 from 192.168.1.100
Welcome to Ubuntu 22.04.3 LTS (GNU/Linux 5.15.0-78-generic x86_64)
tchen@server:~$
SSH Remote Login
ssh username@hostname
ssh username@192.168.1.50
ssh -p 2222 username@server.cod.edu

# Example session:
$ ssh tchen@192.168.1.50
The authenticity of host '192.168.1.50 (192.168.1.50)' can't be established.
Are you sure you want to continue connecting (yes/no/[fingerprint])? yes
tchen@192.168.1.50's password:
Welcome to Ubuntu 22.04.3 LTS
tchen@remoteserver:~$
Security Note: Always use secure protocols like SSH for remote connections. Avoid telnet and other unencrypted protocols.

2. Password Management

Managing user passwords is crucial for system security. Linux provides several commands for password administration.
passwd - Change Password
passwd
# Changes current user's password

sudo passwd username
# Changes another user's password (requires admin privileges)

# Example session:
$ passwd
Changing password for tchen.
Current password:
New password:
Retype new password:
passwd: password updated successfully
Password Policy Commands
passwd -l username # Lock user account
passwd -u username # Unlock user account
passwd -d username # Delete password (dangerous)
passwd -e username # Expire password immediately

# Example output:
$ sudo passwd -l testuser
passwd: password expiry information changed.

$ sudo passwd -u testuser
passwd: password expiry information changed.
Best Practice: Use strong passwords with a combination of uppercase, lowercase, numbers, and special characters.

3. File Commands

File manipulation is fundamental in Linux. These commands help create, copy, move, and manage files and directories.
Basic File Operations
touch filename.txt # Create empty file
cp source.txt dest.txt # Copy file
mv oldname.txt newname.txt # Move/rename file
rm filename.txt # Delete file
mkdir directory_name # Create directory
rmdir directory_name # Remove empty directory
rm -rf directory_name # Remove directory and contents

# Example session:
$ touch newfile.txt
$ ls -l newfile.txt
-rw-r--r-- 1 tchen tchen 0 Jul 03 10:45 newfile.txt

$ cp newfile.txt backup.txt
$ ls
backup.txt newfile.txt

$ mkdir testdir
$ ls -d testdir
testdir/
File Information
file filename.txt # Display file type
stat filename.txt # Display detailed file information
du -h filename.txt # Display file size
find /path -name "*.txt" # Find files by pattern

# Example output:
$ file document.txt
document.txt: ASCII text

$ file script.sh
script.sh: Bourne-Again shell script, ASCII text executable

$ du -h document.txt
4.0K document.txt

$ find /home/tchen -name "*.txt"
/home/tchen/document.txt
/home/tchen/notes.txt
/home/tchen/backup.txt

4. User Account Files (/etc/passwd)

The /etc/passwd file contains essential information about system users. Understanding its structure is crucial for system administration.
Viewing /etc/passwd
cat /etc/passwd
# Display all user accounts

grep "tchen" /etc/passwd
# Search for specific user

# Example output:
$ grep "tchen" /etc/passwd
tchen:x:1001:1001:Tony Chen,,,:/home/tchen:/bin/bash

$ head -5 /etc/passwd
root:x:0:0:root:/root:/bin/bash
daemon:x:1:1:daemon:/usr/sbin:/usr/sbin/nologin
bin:x:2:2:bin:/bin:/usr/sbin/nologin
sys:x:3:3:sys:/dev:/usr/sbin/nologin
sync:x:4:65534:sync:/bin:/bin/sync
passwd File Structure
username:password:UID:GID:GECOS:home_directory:shell

Example:
tchen:x:1001:1001:Tony Chen,,,:/home/tchen:/bin/bash
root:x:0:0:root:/root:/bin/bash
daemon:x:1:1:daemon:/usr/sbin:/usr/sbin/nologin
Security Note: Modern systems use 'x' in the password field, with actual password hashes stored in /etc/shadow for security.

5. Text Processing Commands

Linux provides powerful text processing utilities for viewing, searching, and manipulating file contents.

cat Command

cat - Display File Contents
cat filename.txt # Display entire file
cat file1.txt file2.txt # Display multiple files
cat > newfile.txt # Create file with input
cat file1.txt > file2.txt # Copy file content

# Example output:
$ cat sample.txt
This is line 1
This is line 2
This is line 3

$ cat file1.txt file2.txt
Contents of file1
Contents of file2

head Command

head - Display First Lines
head filename.txt # Display first 10 lines
head -n 5 filename.txt # Display first 5 lines
head -n 20 /etc/passwd # Show first 20 users
head -c 100 filename.txt # Display first 100 characters

# Example output:
$ head -n 3 /etc/passwd
root:x:0:0:root:/root:/bin/bash
daemon:x:1:1:daemon:/usr/sbin:/usr/sbin/nologin
bin:x:2:2:bin:/bin:/usr/sbin/nologin

$ head -n 5 sample.txt
Line 1 of the file
Line 2 of the file
Line 3 of the file
Line 4 of the file
Line 5 of the file

tail Command

tail - Display Last Lines
tail filename.txt # Display last 10 lines
tail -n 15 filename.txt # Display last 15 lines
tail -f /var/log/syslog # Follow file changes (live monitoring)
tail -n +10 filename.txt # Display from line 10 to end

# Example output:
$ tail -n 3 sample.txt
Line 18 of the file
Line 19 of the file
Line 20 of the file

$ tail -f /var/log/syslog
Jul 3 10:45:23 server systemd[1]: Started session-1.scope.
Jul 3 10:45:30 server kernel: [12345.678901] USB disconnect
# (continues to show new lines as they're added)

wc Command

wc - Word Count
wc filename.txt # Display lines, words, characters
wc -l filename.txt # Count lines only
wc -w filename.txt # Count words only
wc -c filename.txt # Count characters only
wc -l /etc/passwd # Count number of users

# Example output:
$ wc sample.txt
20 150 800 sample.txt
# (20 lines, 150 words, 800 characters)

$ wc -l /etc/passwd
45 /etc/passwd

$ wc -w document.txt
250 document.txt

6. Directory Listing Commands

The ls command family provides various ways to list and examine directory contents with different levels of detail.

Basic ls Command

ls - List Directory Contents
ls # List files in current directory
ls /home # List files in specific directory
ls *.txt # List files matching pattern
ls -a # Show hidden files (starting with .)
ls -A # Show hidden files except . and ..

# Example output:
$ ls
Documents Downloads Music Pictures Videos
file1.txt file2.txt script.sh

$ ls -a
. .. .bashrc .profile
Documents Downloads Music Pictures
Videos file1.txt file2.txt script.sh

ls -l Command (Long Format)

ls -l - Detailed File Information
ls -l
# Output format: permissions links owner group size date name

# Example output:
$ ls -l
total 32
-rw-r--r-- 1 tchen tchen 1024 Jul 03 10:30 document.txt
drwxr-xr-x 2 tchen tchen 4096 Jul 03 09:15 projects
-rwxr-xr-x 1 tchen tchen 8192 Jul 02 14:20 script.sh
-rw-r--r-- 1 tchen tchen 256 Jul 01 16:45 notes.txt
drwxr-xr-x 3 tchen tchen 4096 Jun 30 12:00 backup

ls Command Variations

Advanced ls Options
ls -la # Long format including hidden files
ls -lh # Human-readable file sizes
ls -lt # Sort by modification time
ls -lS # Sort by file size
ls -lr # Reverse order
ls -lR # Recursive listing
ls -ld directory_name # List directory itself, not contents

# Example output:
$ ls -lh
total 32K
-rw-r--r-- 1 tchen tchen 1.0K Jul 03 10:30 document.txt
drwxr-xr-x 2 tchen tchen 4.0K Jul 03 09:15 projects
-rwxr-xr-x 1 tchen tchen 8.0K Jul 02 14:20 script.sh

$ ls -ld /home/tchen
drwxr-xr-x 25 tchen tchen 4096 Jul 03 10:45 /home/tchen
Permission Format: The first character indicates file type (- for file, d for directory, l for link), followed by three groups of rwx permissions for owner, group, and others.

7. Command History

Linux maintains a history of executed commands, allowing users to review, repeat, and modify previous commands efficiently.
history - View Command History
history # Display all command history
history 10 # Display last 10 commands
history | grep "ls" # Search history for specific command
history -c # Clear command history

# Example output:
$ history 5
501 ls -l
502 cat /etc/passwd
503 head -n 5 sample.txt
504 wc -l /etc/passwd
505 history 5

$ history | grep "cat"
487 cat sample.txt
502 cat /etc/passwd
520 cat > newfile.txt
History Navigation
!! # Repeat last command
!n # Execute command number n
!string # Execute last command starting with string
!?string # Execute last command containing string
^old^new # Replace 'old' with 'new' in last command

# Example usage:
$ ls -l
$ !!
ls -l # (repeats the ls -l command)

$ !502
cat /etc/passwd # (executes command #502)

$ !cat
cat /etc/passwd # (executes last command starting with 'cat')
History File Management
echo $HISTFILE # Display history file location
echo $HISTSIZE # Display history size limit
history -w # Write current session to history file
history -r # Read history file into current session

# Example output:
$ echo $HISTFILE
/home/tchen/.bash_history

$ echo $HISTSIZE
1000

$ history -w
# (writes current session commands to .bash_history)
Keyboard Shortcuts: Use Up/Down arrows to navigate history, Ctrl+R for reverse search, and Ctrl+G to cancel search.

Process Management
ps # Display running processes
ps aux # Display all processes
top # Display live process information
kill PID # Terminate process by ID
killall process_name # Terminate process by name
Text Search and Manipulation

grep -i "pattern" file # Case-insensitive search
grep -r "pattern" /path # Recursive search
sort filename # Sort file contents
uniq filename # Remove duplicate lines
cut -d: -f1 /etc/passwd # Extract specific fields
